# Network

## 네트워크 개요
### 네트워크 정의와 필요성
- 정의: 두 개 이상의 컴퓨터나 장치들이 서로 연결되어 데이터를 주고받을 수 있도록 구성된 시스템. 
- 필요성: 
  1. 정보 공유:
    - 네트워크를 통해 서로 다른 컴퓨터나 장치들이 데이터를 교환하고 공유 가능. 
    - 예시: 이메일, 클라우드 스토리지.
  2. 자원 공유:
    - 네트워크를 통해 프린터, 하드디스크, 소프트웨어 등과 같은 자원을 여러 사용자와 공유 가능. 
  3. 원거리 통신
  4. 
### 데이터 통신의 기본 구성 요소
#### 1. 송신자
- 데이터를 보내는 장치나 시스템.
- 예시: 컴퓨터, 스마트폰, 서버.
#### 2. 수신자
- 송신자가 전송한 데이터를 받는 장치나 시스템. 
- 예시: 컴퓨터, 스마트폰, 서버.
#### 3. 전송 매체
- 송신자와 수신자 간에 데이터를 전달하는 물리적 경로.
- 예시: 적외선, 전화선, 케이블 등.
#### 4. 프로토콜
- 송신자와 수신자 간의 데이터 통신을 원활하게 진행하기 위해 정해진 규칙이나 표준. 
- 예시:  TCP/IP, HTTP, FTP, SMTP 등.

### 네트워크 모델 (Network Models)
#### OSI 7계층 모델- Physical, Data Link, Network, Transport, Presentation, Applicatioin
1. 물리 계층(Physical Layer):
   - 비트 단위로 데이터를 전송하며, 전기 신호나 광 신호로 변환하여 전송.
   - 예: 케이블, 커넥터 등. 
2. 데이터 링크 계층(Data Link Layer):
  - 물리 계층에서 전송된 데이터를 프레임으로 묶고 오류를 감지하며, 데이터의 흐름을 제어,
   - 예: MAC 주소, 이더넷 등.
3. 네트워크 계층(Network Layer):
   - 데이터를 목적지까지 전송할 수 있도록 라우팅을 담당. 
   - IP 주소를 통해 전달 될 수 있도록 경로 설정 역할.
   - 예: IP, ICMP, 라우팅 등. 
4. 전송 계층(Transport Layer):
   - 데이터의 흐름 제어, 오류 복구, 그리고 세그먼트화된 데이터의 전송을 담당.
   - 두 호스트 간의 연결을 관리하고, 데이터가 올바르게 전달되도록 보장. 
   - 예: TCP, UDP 등. 
5. 세션 계층(Session Layer):
   - 두 컴퓨터 간의 연결을 설정, 유지, 종료하는 역할을 함.
   - 세션을 관리하고 동기화하여 데이터 전송 흐름을 제어.
   - 예: RPC, NetBIOS 등.
6. 표현 계층(Presentation Layer):
   - 데이터의 형식이나 구조를 변환하여 송신자와 수신자 간에 이해할 수 있도록 함.
   - 예: JPEG, GIF, ASCII, SSL/TLS (보안 계층) 등.
7. 응용 계층(Application Layer):
   - 사용자가 직접 사용하는 소프트웨어와 관련된 계층.
   - 웹 브라우저, 이메일, 파일 전송 등의 서비스가 이 계층에서 동작. 
   - 예: HTTP, FTP, SMTP, DNS 등.
#### TCP/IP 모델
- 정의: TCP/IP 모델은 인터넷 및 네트워크 통신의 표준 모델로, 4개의 계층으로 구성.
- 특징:
  - 계층별로 분리된 기능을 통해 네트워크의 관리와 이해가 용이.
  - 하위 계층에서 상위 계층을 지원.
  - 각 계층은 데이터를 캡슐화하여 전달. 
- 장점: 계층화된 구조로 인해 네트워크 관리 및 문제 해결이 용이.
- 단점: 과도한 캡슐화가 발생할 수 있어 성능 저하가 발생할 수 있음.
### 네트워크의 종류 (Type of Networks)
#### 1. LAN
- 정의: 제한된 물리적 범위 내에서 장치들을 연결.
- 특징:
  - 빠른 데이터 전송 속도.
  - 물리적 범위가 제한적.
  - 설정과 유지가 간단.
  - 장치들이 중앙 장치에 연결되어 데이터를 주고 받음.
- 장점: 설치가 용이하고 빠른 데이터 전송 속도를 제공. 
- 단점: 대규모 네트워크에 적합하지 않다. 
#### 2. WAN
- 정의: 광범위한 지역을 연결하는 네트워크로, 다양한 지리적 위치를 연결.
- 특징:
  - 긴 거리와 고비용
  - 낮은 데이터 전송 속도
- 장점: 넓은 범위에서 연결 가능.
- 단점: 높은 설치 비용과 낮은 데이터 전송 속도.
#### 3. MAN
- 정의: 도시 규모의 네트워크.
- 특징:
  - 도시 내에서 운영.
  - LAN과 WAN의 중간 규모.
- 장점: 도시 범위 내에서 고속 연결을 제공.
- 단점: 설치 및 유지 비용이 LAN보다 높음. 
#### 4. PAN
- 정의: 개인 장치 간의 무선 연결을 지원하는 네트워크.
- 특징:
  - 매우 짧은 거리에서 연결.
  - 개인적인 기기들 간의 데이터 전송.
- 장점: 설정이 간편하고 저전력 소비.
- 단점: 연결 범위가 매우 제한적. 
### 네트워크 토폴로지 (Network Topology)
#### 스타, 버스, 링, 메시, 트리
#### 1. 스타
- 정의: 모든 장치가 중앙 허브나 스위치에 연결되어 있는 네트워크 구조.
- 특징:
  - 중앙 집중형.
  - 허브나 스위치가 장애가 되면 전체 네트워크가 영향을 받음.
- 동작:
  - 각 장치는 중앙 장치로 데이터를 전송하고, 중앙 장치는 이를 다른 장치로 전달. 
- 장점: 장치 추가가 간편.
- 단점: 중앙 장치에 문제가 생기면 전체에 영향.
#### 2. 버스
- 정의: 모든 장치가 하나의 공통된 전송 매체를 통해 연결되는 네트워크 구조.
- 특징:
  - 설치가 간단하고 비용이 적게 든다.
  - 장애 발생 시 전체 네트워크가 영향을 받을 수 있다.
- 동작:
  - 모든 장치가 하나의 버스를 통해 데이터를 전송하고, 이를 다른 장치들이 수신. 
- 장점: 설치가 간편하고 비용이 적음.
- 단점: 버스에 장애가 발생하면 모든 장치에 영향을 미침.
#### 3. 링
- 정의: 장치들이 원형으로 연결되어 데이터를 한 방향으로 순환시키는 네트워크.
- 특징:
  - 원형 경로를 따라 한 방향으로 전송됨. 
  - 장치가 추가되면 네트워크 성능이 감소할 수 있음.
- 동작:
  - 각 장치는 이전 장치로부터 데이터를 받고, 이를 다음 장치로 전달. 
- 장점: 데이터 전송이 일정한 속도로 이루어짐.
- 단점: 한 장치가 장애를 일으킬 경우 전체 네트워크가 영향을 받음.
#### 4. 메시
- 정의: 모든 장치가 서로 연결되어 있는 네트워크 구조.
- 특징:
  - 여러 경로를 통해 데이터를 전달.
  - 하나의 경로에 문제가 생기면 다른 경로를 사용.
- 장점: 고속, 높은 신뢰성.
- 단점: 설치 및 유지 보수가 매우 높음. 
#### 5. 트리
- 정의: 스타와 버스를 결합한 형태의 네트워크 구조로, 계층적 구조를 형성.
- 특징: 
  - 계층적 구조로 확장이 용이.
  - 상위 계층 장치가 장애를 일으키면 하위 장치에 영향을 미침.
- 동작:
  - 상위 계층 장치가 하위 장치들로 데이터를 전달.
- 장점: 확장이 용이하고 관리가 편리.
- 단점: 상위 계층의 장애 시 전체 네트워크가 영향을 받음. 
## 데이터 통신 및 전송 기술 (Data Communication and Transmission Technologies)
### 데이터 표현과 신호
#### 디지털 데이터
- 정의: 연속적이지 않고, 일정한 크기의 단위로 나누어져 있는 데이터.
#### 1. 이진 신호
- 정의: 디지털 데이터의 기본적인 표현 방식으로, 0과 1의 두 가지 값만을 사용.
- 특징:
  - 두 가지 상태만을 가짐.
  - 아날로그 신호보다 잡음에 강해 전송 오류가 적음.
#### 2. 샘플링
- 정의: 아날로그 신호를 일정한 시간 간격으로 측정하여 디지털 값으로 변환하는 과정.
- 특징:
  - 주파수가 높을수록 신호를 더 정확하게 디지털화할 수 있음.
- 장점: 데이터를 압축하고, 처리하거나 전송하는 데 있어 효율적.
- 단점: 샘플링 주파수가 낮을 경우 아날로그 신호의 중요한 세부 사항을 놓칠 수 있음. 
#### 아날로그 신호
- 정의: 시간에 따라 변화하는 연속적인 값을 가지는 신호. 
- 특징:
  - 값이 연속적으로 변화함.
  - 일정한 시간 동안 가능한 모든 값을 가질 수 있음. 
#### 1. 주파수
- 정의: 신호의 변화 속도를 나타내며, 일정한 시간 동안 신호가 반복되는 횟수를 의미.
- 특징:
  - 높은 주파수는 신호가 빠르게 변화하는 것을 의미.
  - 주파수가 높을수록 더 많은 정보를 전송할 수 있는 잠재력을 가짐. 
#### 2. 진폭
- 정의: 파형의 최댓값과 최솟값 사이의 차이.
- 특징:
  - 진폭이 클수록 신호의 세기가 강함.
  - 진폭이 크면 소리가 커지고, 진폭이 작으면 소리가 작다. 
#### 3. 위상
- 정의: 주기적인 신호의 특정 지점이 다른 신호에 대해 얼마나 차이가 나는지를 나타냄.
- 특징:
  - 두 개의 파형이 같은 주파수와 진폭을 가지더라도, 위상이 다르면 신호의 시작점이 다름. 
  - 위상이 0도일 경우 두 신호가 일치.

### 부호화와 변조
1. 라인 부호화
  - 정의: 디지털  데이터를 특정한 신호로 변환하는 과정을 의미.
  - 특징:
      - 신호의 전송 방식에 대한 규칙 정의.
  - 장점: 구현이 비교적 간단.
  - 단점: 긴 연속적인 1 또는 0이 발생할 때 동기화가 어려워질 수 있음. 
2. 주파수 변조
  - 정의: 아날로그 신호에서 반송파의 주파수를 변조하여 정보를 전달하는 방식.
  - 특징:
    - 변조된 신호의 주파수가 변할 때마다 다른 정보를 나타냄.
  - 장점: 진폭 변조에 비해 잡음에 강한 특성을 가짐. 
  - 단점: 주파수 변조는 대역폭을 많이 차지. 
3. 진폭 변조
  - 정의: 아날로그 신호에서 반송파의 진폭을 변조하여 정보를 전달.
  - 특징:
    - 신호의 진폭이 변하면서 정보를 전달. 
    - 원래 신호의 진폭을 반송파의 진폭과 결합하여 변조. 
  - 장점: 구현이 간단하고 저비용의 장비로 구현 가능.
  - 단점: 진폭 변조는 잡음에 민감. 
4. 위상 변조
  - 정의: 아날로그 신호에서 반송파의 위상을 변조하여 정보 전달.
  - 특징:
    - 반송파의 위상을 변동시켜 정보를 표현. 
    - 원래 신호의 위상을 반송파의 위상에 결합하여 변조.
  - 장점: 진폭 변조와 비교해 잡음에 강하고 신호 품질이 높다.
  - 단점: 일정한 대역폭을 요구한다.
#### 다중화 기술
#### 1. TDM
- 정의: 전송 매체의 대역폭을 시간적으로 나누어 사용하는 방식. 
- 특징:
  - 하나의 채널을 시간적으로 나누어 사용.
  - 모든 사용자가 동일한 주파수를 공유하지만, 서로 다른 시간 슬롯에서 데이터를 전송.
- 동작:
  1. 데이터 흐름이 시간 슬롯에 따라 순서대로 전송됨.
  2. 각 슬롯은 정해진 순서로 순환하며, 충돌 없이 데이터를 전송.
  3. 동기식 TDM은 고정된 시간 슬롯을, 비동기식 TDM은 필요에 따라 시간 슬롯을 할당.
- 장점: 사용자에게 고유의 시간 슬롯을 제공하여 충돌을 방지.
- 단점: 데이터를 전송하지 않을 경우에도 슬롯이 낭비. 
#### 2. FDM
- 정의: 전송 매체의 대역폭을 주파수 대역으로 나누어 사용하는 방식.
- 특징:
  - 전송 매체의 주파수 대역폭을 여러 개의 작은 대역으로 나눈다.
  - 각 사용자에게 고유의 주파수 대역을 할당. 
- 동작:
  1. 데이터 흐름은 각기 다른 주파수 대역으로 변조.
  2. 모든 신호가 동시에 전송 매체 통과.
  3. 수신 측에서 각 주파수 대역의 데이터를 분리하여 복조.
- 장점: 여러 사용자가 동시에 데이터 전송 가능. 
- 단점: 주파수 변조와 복조 장치가 필요. 
#### 3. CDM
- 정의: 전송 매체를 코드 시퀀스를 기반으로 공유하는 방식.
- 특징:
  - 모든 사용자가 동일한 시간과 주파수 대역 사용.
  - 고유한 스프레드 스펙트럼 코드를 사용하여 데이터를 구분.
- 동작:
  1. 각 데이터는 고유한 코드로 변조.
  2. 전송 매체를 통과하는 동안, 각 데이터는 고유 코드에 의해 스펙트럼이 확장.
  3. 수신 측에서 고유 코드를 사용해 데이터를 복원. 
- 장점: 코드가 고유하여 신호를 쉽게 도청하거나 방해하기 어려움. 
- 단점: 코드 생성 및 관리가 복잡. 

### 전송 방식
#### 직렬 전송과 병렬 전송
#### 1. 직렬 전송
- 정의: 데이터를 한 번에 한 비트씩 전송하는 방식.
- 특징:
  - 데이터 비트를 한 줄로 직렬로 전송.
  - 비교적 적은 수의 전송 라인을 사용.
- 동작:
  1. 데이터가 직렬로 변환되어 전송.
  2. 수신 측에서 직렬 데이터를 다시 병렬로 복원.
  3. 클럭 신호를 사용해 전송의 동기화를 유지.
- 장점: 긴 거리 전송 가능.
- 단점: 속도가 느리고 병렬로 변환하는 복원 과정이 필요하다. 
- 활용: USB, RS-232.
#### 2. 벙렬 전송
- 정의: 데이터를 여러 비트씩 동시에 전송하는 방식.
- 특징:
  - 여러 비트를 병렬로 전송하여 속도를 높임. 
  - 여러 전송 라인이 필요.
- 동작:
  1. 데이터가 병렬 형식으로 전송.
  2. 수신 측에서 그대로 처리하거나 다시 직렬로 변환.
  3. 동기화 신호를 사용해 전송 선로의 데이터를 조정.
- 장점: 직렬 전송보다 속도가 빠름. 
- 단점: 여러 전송 선로로 인해 신호 간 간섭과 타이밍 문제 발생 가능. 
- 활용: DDR 메모리, 컴퓨터 내부 데이터 버스.
#### 동기식 전송과 비동기식 전송
#### 1. 동기식 전송
- 정의: 송신 측과 수신 측이 동일한 클럭 신호를 사용해 데이터를 전송하는 방식. 
- 특징:
  - 송신 측과 수신 측 간의 정확한 클럭 동기화 필요.
  - 데이터를 블록 단위로 처리하여 전송.
  - 전송 효율이 높음. 
- 동작:
  1. 송신 측과 수신 측이 동일한 클럭 신호를 공유.
  2. 시작 비트와 정지 비트 없이 연속적으로 데이터 전송. 
  3. 클럭 신호 기준으로 수신 측에서 해석. 
- 장점: 
  - 비동기식 전송보다 속도 빠름. 
  - 시작 비트와 정지 비트를 사용하지 않아 오버헤드가 적음. 
- 단점: 클럭 신호를 송신 측과 수신 측에서 정확히 일치시켜야 함. 
- 활용: WAN, LAN, HDLC

#### 2. 비동기식 전송
- 정의: 송신 측과 수신 측이 클럭 신호를 공유하지 않고, 데이터 단위를 개별적으로 전송하는 방식. 
- 특징:
  - 송신 측과 수신 측 간의 클럭 동기화가 필요하지 않음. 
  - 데이터를 개별 문자 또는 바이트 단위로 전송. 
- 동작:
  1. 각 데이터 단위에 시작 비트와 정지 비트를 추가. 
  2. 송신 측은 데이터 단위를 개별적으로 전송. 
  3. 수신 측은 시작 비트와 정지 비트를 기반으로 데이터를 해석. 
- 장점: 저비용으로 구현 가능. 
- 단점: 동기식 전송에 비해 느림. 
- 활용: 키보드 입력, RS-232 통신.
